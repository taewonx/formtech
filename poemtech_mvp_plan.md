# FormTech MVP 개발 계획 및 클라이언트 AI 바이오메카닉스 텔레메트리 아키텍처

1인 창업의 핵심은 **'서버 유지비(Burn rate)의 최소화'**와 **'가장 뾰족한 하나의 기능에 집중하는 것'**입니다. FormTech은 운동 방법을 참견하는 '코치'가 아닙니다. 헤비 리프터들이 쌓아 올린 스트렝스를 단 1%의 유실도 없이 바벨에 전달하도록 돕는 프라이빗 **바이오메카닉스 텔레메트리 센서(Biomechanics Telemetry Analyzer)**를 유저의 기기 자원만을 활용하는 클라이언트 단 AI로 구현합니다.

---

## 1. 아키텍처 개요 (Architecture Overview)

```
[유저 웹캠/영상] ──▶ [Browser (React + MediaPipe)] ──▶ (실시간 역학 연산 및 UI 텔레메트리 렌더링)
                              │
                              ▼ 
                 [Local Storage (Web API)] ──▶ (로컬 브라우저 내 세션 텔레메트리 로그 저장)
```
*서버 데이터베이스(Supabase 등) 없이 100% 클라이언트 환경에서 동작하도록 구현하여 유지비 0원 달성 및 프라이버시 보호.*

---

## 2. 타겟 시장 및 페인 포인트 (Target Market & Pain Point)

### 2.1 핵심 페인 포인트
1. **역학적 힘의 누수 (Energy Leakage)**: 스쿼트, 데드리프트(SBD) 등 고중량 운동 시 미세한 골반 축 무너짐(스쿼트 벗윙크)이나 요추 굴곡(데드리프트 척추 굽음)으로 인해 힘의 작용선이 어긋남.
2. **참견식 코칭에 대한 거부감**: 높은 숙련도를 갖춘 헤비 리프터들은 훈계가 아닌, 정밀 데이터(각도 편차 등)를 건네주는 계측 도구를 강력히 요구함.
3. **텔레메트리 데이터의 지연**: 녹화본 확인으로는 즉각적인 교정 피드백을 전달받을 수 없음.

### 2.2 타겟 유저
* **Primary 타겟**: 파워리프팅, 스트렝스 트레이닝을 수행하는 SBD 헤비 리프터 및 코어 리프터층.

## 3. 기술 스택 (Technology Stack)

* **Client-side AI & Vision**: `TensorFlow.js` + `MediaPipe Pose`
  * *이유*: 서버로 영상을 전송하지 않고 유저의 브라우저 단에서 직접 33개의 관절 좌표(Landmarks)를 추출. 프라이버시 문제와 서버 비용을 완벽하게 해결.
* **Frontend Framework**: `React.js` (Vite)
  * *이유*: 실시간 비디오 스트림 위에 스켈레톤(Skeleton) 레이어를 얹고, 상태 관리(카메라/영상 전환, 결과 저장 등)를 쉽게 구축.
* **Styling**: Vanilla CSS + Tailwind CSS (Dark/Neon 펑크 테마)
  * *이유*: Tailwind를 통해 빠른 생산성을 유지하면서도, 커스텀 CSS를 조합하여 프리미엄하고 역동적인 테마를 완벽하게 구현. (충돌 나던 기존 유틸리티 클래스는 제거)
* **Storage**: `Browser LocalStorage`
  * *이유*: 초기 MVP에서는 유지비 및 복잡성 감소를 위해 외부 백엔드를 배제하고 기기 내부에 데이터를 기록하여 오프라인 앱 수준의 빠른 속도 제공.

## 4. 구현된 핵심 기능 (MVP Features)

| 기능 분류 | 상세 기능 | 비고 |
| :--- | :--- | :--- |
| **강제 가이드라인 Flow** | 4가지 필수 촬영 가이드라인 체크박스를 모두 확인해야만 카메라/업로드 UI가 활성화되는 UX 구축 | Garbage In 방지 |
| **실시간 캡처 & 영상 업로드** | 웹캠 실시간 폼 체크 및 기기 내 비디오 파일(.mp4, .mov) 업로드 후 분석 기능 지원 | 100% 로컬 프로세싱 |
| **관절 추적 및 시각화** | MediaPipe 기반 스켈레톤 렌더링 및 운동 종목별(스쿼트/데드리프트) 타겟 부위 트래킹 | 클라이언트 연산 |

| **로컬 세션 리포트** | 분석 완료 후 '내 운동 기록' 영역에 결과(총 횟수, 성공률 등)를 로컬스토리지 기반으로 영구 저장 | |

## 5. UI/UX 개선 세부 내역

개발 중 반영된 주요 UI/UX 개선 사항:
1. **강제 스텝형 UI 도입**: 촬영 구도(90도 측면, 타이트한 옷 등)를 지키지 않을 경우 AI 정확도가 크게 떨어지기 때문에, 분석기를 보여주기 전 **가이드라인 체크를 필수로 완료(Step 1)** 하도록 강제.
2. **아코디언(접기/펼치기) 가이드라인**: 가이드라인 확인 후 버튼을 누르면 패널이 화면에서 사라지지 않고 얇은 요약 바로 접히게 되어, 언제든 다시 펼쳐서 가이드를 참고할 수 있도록 편의성 극대화.
3. **디자인 테마 및 반응형**: 칙칙한 디자인을 버리고, FormTech 브랜드에 맞는 다이나믹한 다크/네온 테마(보라색 포인트, 마이크로 애니메이션 등)를 도입. 화면 높이가 작아질 때 글씨가 겹치던 문제와 가이드 이미지가 잘리던 버그 완벽 해결.
4. **브랜드 에셋 업데이트**: 'Poemtech'에서 'FormTech'으로 이름 변경. 기본 이모지 로고에서 사용자가 직접 디자인한 **FormTech LOGO(white).png**를 파비콘 및 네비게이션 로고에 적용.

## 6. 비디오 품질 예외 처리 방안 (Garbage In, Garbage Out 예외 처리)

### 6.1 기술적 방어: MediaPipe '신뢰도(Confidence Score)' 활용
* 주요 관절의 신뢰도가 0.5 미만으로 떨어질 경우, 오버레이 경고 UI("인식이 어려워요")를 노출하고 측정을 일시 정지(또는 피드백 보류)시킵니다.

### 6.2 UX의 제약 조건 명확화 (좁고 깊게)
* **단일 화각 강제**: MVP 단계에서는 "측면 90도에서 촬영한 영상만 분석 가능합니다"로 범위를 제한.
* **가이드라인 필수 확인 로직**: 체크리스트를 누르지 않으면 카메라 접근 버튼 자체가 활성화되지 않도록 프론트엔드 레벨에서 접근을 차단하여 불량 데이터를 원천 차단했습니다.
